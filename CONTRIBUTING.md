# Contributing to Good Jobs Frontend

First off, thank you for considering contributing to Good Jobs! It's people like you that make Good Jobs such a great platform for the North-East India community.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](../../issues) first. If it's a new idea or bug, please open a new issue.

## Pull Request Process

1. **Fork the repo** and create your branch from `master`.
2. **Commit your changes**. Use clear and descriptive commit messages (e.g., `feat: add new filter to jobs` or `fix: resolve mobile nav overlap`).
3. **Test your changes** to ensure they look good on both Desktop and Mobile browsers. Good Jobs relies heavily on a responsive UI.
4. **Update the README.md** if your changes add new dependencies, setup steps, or drastically alter the platform's features.
5. **Issue a Pull Request**. Ensure the PR description clearly details the problem and your solution.

## Code Style

- **HTML/CSS:** Please write semantic HTML. Use the variables defined in `index.css` (e.g., `var(--primary)`) rather than hard-coding hex values. Avoid Tailwind or other frameworks unless pre-approved; we use Vanilla CSS.
- **JavaScript:** We use standard Vanilla ES6+ JavaScript. Keep components modular and ensure `main.js` functions remain clean and well-documented.

## Code of Conduct

Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.
